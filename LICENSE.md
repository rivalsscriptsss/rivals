## 🛠️ Troubleshooting Common Issues

**❌ Script fails to load after injection**
The executor may have lost connection to the game process during attachment. This typically happens when the game is still loading assets.
**Solution:** Wait until you are fully loaded into the lobby before attaching. Restart the executor if the issue persists with your **Rivals Script** build.

**❌ GUI overlay does not appear on screen**
Some executor versions render overlays on a secondary display or behind the game window.
**Solution:** Press the `toggle_gui` keybind (default `RightShift`) twice. If still invisible, check `rivals_config.json` for `"gui_visible": false` and change it to `true`.

**❌ Blue Lock Rivals Script Mobile crashing on launch**
Older Android devices may lack the required WebView version for the floating menu.
**Solution:** Update Android System WebView from the Play Store. Make sure your **Blue Lock Rivals Script Mobile** build matches your device architecture (ARM64 vs ARM32).

**❌ Aim Assist not tracking targets correctly**
This usually means the `fov_radius` is set too low or the game updated its character hitbox data.
**Solution:** Increase `fov_radius` to `140` in the config and update the **Aim Assist Rivals Script** module to the latest version from the repository.

**❌ Anti Katana not activating during matches**
The module requires line-of-sight calculation which can fail if your frame rate drops below 30 FPS.
**Solution:** Lower your in-game graphics settings or reduce the **Rivals Script Anti Katana** `reaction_delay_ms` to `30` for faster response times.

**❌ Skin Change showing default appearance**
The **Rivals Script Skin Change** module applies client-side only by default.
**Solution:** This is expected behavior. Other players see your original skin unless the server-side flag is enabled in configuration.
